

# ベクトル検索RAGシステム実装計画

会話履歴をベクトル化し、意味的な類似検索で関連する過去の相談を取得できるシステムを構築します。

## 現状の課題

現在の実装は直近10件のインサイトをそのまま取得しているため：
- 関連性の低い過去データも含まれる
- データ量が増えると処理が重くなる
- ユーザーの現在の相談内容と関連する過去情報を見つけられない

## 新しいRAGアーキテクチャ

```text
┌─────────────────────────────────────────────────────────────┐
│                    セッション終了時                           │
├─────────────────────────────────────────────────────────────┤
│  会話内容 → AI要約/キーワード抽出 → ベクトル化 → DB保存       │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                    新しいセッション開始時                     │
├─────────────────────────────────────────────────────────────┤
│  ユーザーの質問 → ベクトル化 → 類似検索 → 関連コンテキスト取得  │
└─────────────────────────────────────────────────────────────┘
```

## 実装ステップ

### 1. pgvector拡張の有効化とテーブル更新

`conversation_insights`テーブルに`embedding`カラムを追加し、ベクトル検索用のインデックスを作成します。

```text
conversation_insights テーブル
├── 既存カラム（summary, keywords, topics など）
└── 新規: embedding vector(384)  ← 要約テキストのベクトル表現
```

### 2. ベクトル類似検索用のPostgres関数

ユーザーのクエリベクトルと保存済みベクトルの類似度を計算し、関連する過去の相談を返す関数を作成します。

```text
match_conversation_insights(
  query_embedding: vector(384),
  match_threshold: float,
  match_count: int,
  user_id: uuid
) → 関連インサイト一覧
```

### 3. エッジ関数の更新

**extract-conversation-insights関数の更新**:
- 会話要約を生成後、Lovable AIモデルを使用してベクトル（embedding）を生成
- ベクトルと一緒にDBに保存

**新規: search-relevant-insights関数**:
- ユーザーの現在の質問/話題をベクトル化
- 類似検索で関連する過去インサイトを取得
- 整形してコンテキストとして返却

### 4. フロントエンド統合

`useConversationInsights`フックを更新：
- `getRelevantContext`を類似検索ベースに変更
- セッション開始時やメッセージ送信時に関連コンテキストを取得

## 技術詳細

### 使用するEmbeddingモデル

Lovable AIゲートウェイを通じてembeddingを生成します。

### ベクトル次元

384次元のベクトルを使用（効率的でかつ十分な精度）

### 類似度計算

コサイン距離（`<=>`演算子）を使用して意味的な類似度を計算

### インデックス

IVFFlatインデックスを使用してクエリを高速化

## 処理フロー詳細

### セッション終了時
1. 会話内容をAIで分析（既存処理）
2. 要約テキストからembeddingを生成（新規）
3. インサイト + embeddingをDBに保存（更新）

### 新規相談時
1. ユーザーの最初の発言/質問をembedding化
2. DBで類似ベクトル検索（cosine similarity）
3. 上位5件の関連インサイトを取得
4. 占い師AIのコンテキストとして注入
